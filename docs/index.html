Open-source VPS sustainability tool

VPS Energy Monitor

Estima el consumo energético y las emisiones CO2e de un VPS Ubuntu con métricas locales de CPU, RAM y una metodología transparente.

Qué hace

Un recolector ligero para VPS Linux

Monitorea CPU y RAM Guarda muestras locales en SQLite usando datos de /proc.
Corre en background Incluye una unidad systemd para iniciar con Ubuntu.
Genera reportes Entrega estimaciones por día, semana o mes desde la terminal.
Calcula CO2e Convierte energía estimada con un factor eléctrico regional.
Fórmula

Modelo de cálculo

Para un VPS sin GPU, la estimación usa potencia de CPU, RAM y PUE.

Energy kWh =
hours x ((vCPU x W_per_vCPU x CPU_usage)
+ (RAM_GB x W_per_GB_RAM)) / 1000 x PUE

CO2e =
Energy kWh x grid_emission_factor
Configuración

Valores que debes ajustar

La precisión depende menos del script y más de los factores que declares. Usa primero datos del proveedor del VPS o del data center. Si no existen, usa fuentes públicas y deja documentado de dónde salió cada supuesto.

W_PER_VCPU Potencia estimada de una vCPU cuando trabaja al 100%.
W_PER_GB_RAM Potencia estimada por cada GB de memoria RAM asignada o usada.
PUE Multiplicador de energía del centro de datos: energía total / energía IT.
KGCO2E_PER_KWH Intensidad de carbono de la electricidad donde corre el VPS.

W_PER_VCPU

15

Representa cuántos watts se asignan a una vCPU a carga completa. Si conoces el procesador físico, toma su TDP del fabricante y divídelo entre el número de núcleos. En VPS normalmente no se publica el host exacto, así que conviene usar un rango razonable y explicar el supuesto.

  • Más conservador: 10 a 20 W por vCPU.
  • Default práctico: 15 W por vCPU.
  • Mejor dato: documentación técnica del CPU o del proveedor.

W_PER_GB_RAM

0.4

La memoria consume energía aunque el workload no la esté usando intensamente. Por eso, para reportes prudentes, RAM_MODE=allocated suele ser más defendible que usar solo RAM usada.

  • Green Algorithms usa una referencia cercana a 0.3725 W/GB.
  • El default 0.4 redondea ese valor para simplificar.
  • Usa otro valor si tu proveedor publica especificaciones del hardware.

PUE

1.2

PUE incluye refrigeración, distribución eléctrica, UPS, iluminación y otros consumos de infraestructura. Un PUE de 1.2 significa que por cada 1 kWh usado por equipos IT, se agregan 0.2 kWh de overhead.

  • Mejor dato: PUE publicado por el proveedor o región del data center.
  • Hiperscalers eficientes pueden estar cerca de 1.1 a 1.3.
  • Si no sabes el proveedor o instalación, 1.5 a 1.6 es una referencia prudente de industria.

KGCO2E_PER_KWH

0.05

Es el factor más sensible del cálculo. Primero identifica la región real del VPS, no la ubicación del usuario. Después busca el factor eléctrico location-based de esa red o país y conviértelo a kgCO2e/kWh.

  • Si la fuente está en gCO2e/kWh, divide entre 1000.
  • Si la fuente está en kgCO2e/kWh, úsala directamente.
  • Si la fuente está en tCO2e/MWh, numéricamente equivale a kgCO2e/kWh.

RAM_MODE

allocated

allocated usa toda la RAM asignada al VPS. used usa solo la RAM observada en Linux. Para estimaciones públicas o comparables, allocated evita subestimar memoria reservada; para seguimiento operativo interno, used puede mostrar mejor la variación del workload.

Fuentes útiles para elegir factores

  • Cloud Carbon Footprint methodology: referencia abierta para estimar energía y CO2e en servicios cloud con uso, PUE y factores de red.
  • Green Algorithms: metodología de cómputo que usa tiempo, núcleos, memoria, PUE e intensidad de carbono.
  • Green Software Foundation Impact Framework: guía para transformar utilización de CPU en watts, energía y carbono.
  • Uptime Institute Global Data Center Survey 2025: referencia de industria para PUE promedio cuando no tienes el dato del proveedor.
  • EPA eGRID: factores eléctricos por subregión en Estados Unidos.
  • IEA Emissions Factors: factores de emisiones eléctricas por país para comparaciones internacionales.
Instalación

Uso rápido

1

Clona el repositorio

git clone https://github.com/garcelomarcia/vps-energy-monitor.git
cd vps-energy-monitor
2

Ejecuta el instalador

sudo ./install.sh
3

Edita la configuración

sudo nano /etc/vps-energy.conf
4

Consulta reportes

vps-energy status
vps-energy report --period day
vps-energy report --period week
vps-energy report --period month

Limitación

Estimación, no medidor físico

En un VPS, el hardware real se comparte con otros usuarios. Por eso el resultado debe entenderse como una estimación basada en recursos virtuales y uso observado.

Para reportes formales, usa datos oficiales del proveedor cloud cuando estén disponibles.
